Nikita Babych — Key Technical Insights
Core Innovation: PowerTransform Pseudo-Labeling
After generating pseudo-label probabilities from the ensemble "teacher", apply a power transform pʸ (power scaling) to sharpen/recalibrate the soft labels before the next student training round. This prevents probability collapse across 4 iterations and maintains discriminative signal. Combined with MixUp, StochasticDepth regularization creates a robust noisy student framework.
SoftAUCLoss KEY
Direct AUC
Computes pairwise probability differences and applies log-loss. Directly optimizes the competition metric (macro ROC-AUC) instead of a proxy. Supports soft pseudo-labels. Resistant to overfitting in later training rounds.
# Pairwise AUC approximation
# loss = E[log(sigmoid(p_pos - p_neg))]
# Works with soft label targets

2.1 CrossEntropyLoss → SoftAUCLoss Progression KEY
Start with CE loss (step 1), then switch to SoftAUCLoss in later rounds (step 2). This matches what 2025 1st place did.
Step 1 (sed-b0-v6): CrossEntropy with label smoothing 0.05 — known +0.044 over BCE (2024 1st place)
Step 2 (sed-b0-v7+): Switch to SoftAUCLoss for pseudo-label rounds — directly optimizes competition metric
# Phase 1: CE with label smoothing
loss = F. cross_entropy (logits, hard_labels, label_smoothing= 0.05 )
# Phase 2: SoftAUC for later rounds (pseudo-label training)
def soft_auc_loss (pred, target, margin= 1.0 ):
# Pairwise positive-negative difference → log-sigmoid loss
pos = pred[target > 0.5 ]; neg = pred[target 0.5 ]
diffs = pos. unsqueeze ( 1 ) - neg. unsqueeze ( 0 )
return -F. logsigmoid (diffs * margin). mean ()
⚠ CE Risk: Most 5s clips in BirdCLEF are single-species dominant — CE works here. But test on holdout before committing. If CE hurts, fall back to BCE+Focal combo (2nd place approach).

4.1 OOF Pseudo-Labeling Strategy (2025 2nd Place) NEW
Generate pseudo-labels using out-of-fold models to avoid label leakage. Refresh labels every round with the stronger model.
# OOF pseudo-labeling loop
for fold in range( 5 ):
val_model = load_model (fold_checkpoint[fold])
val_soundscapes = soundscapes_for_fold[fold]
preds = val_model. predict (val_soundscapes)
# Filter: max prob >= 0.5, suppress
mask = preds. max (axis= 1 ) >= 0.5
preds[preds 0.1 ] = 0
pseudo_labels[fold] = preds[mask] # soft targets

5.3 Optuna Ensemble Weight Search NEW
Use Optuna to find optimal per-model weights on holdout set. 2025 2nd place used this for their 15-model ensemble. Simple to implement with existing holdout eval.
import optuna
def objective (trial):
w_perch1 = trial. suggest_float ( 'w_perch1' , 0.5 , 2.0 )
w_perch2 = trial. suggest_float ( 'w_perch2' , 0.5 , 2.0 )
w_sed = trial. suggest_float ( 'w_sed' , 1.0 , 4.0 )
ensemble = (w_perch1*p1 + w_perch2*p2 + w_sed*p_sed) / (w_perch1+w_perch2+w_sed)
return roc_auc_score (y_holdout, ensemble)
study = optuna. create_study (direction= 'maximize' )
study. optimize (objective, n_trials= 200 )
